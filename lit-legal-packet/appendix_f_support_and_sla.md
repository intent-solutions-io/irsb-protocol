## F.6 SEVERITY DEFINITIONS

### Critical Severity
- **Definition:** Skill is completely non-functional
- **Impact:** All users unable to use affected skill
- **Examples:**
  - Skill fails to load or execute
  - Complete loss of core functionality
  - Security vulnerability requiring immediate action

### High Severity
- **Definition:** Major functionality impaired
- **Impact:** Significant feature unavailable; workaround difficult
- **Examples:**
  - Key workflow broken
  - Output consistently incorrect
  - Performance degradation preventing normal use

### Normal Severity
- **Definition:** Minor functionality issue
- **Impact:** Workaround available; limited user impact
- **Examples:**
  - Documentation error
  - Edge case handling issue
  - Minor UI/output formatting issues

## F.7 SUPPORT PROCESS

### Submitting Support Requests

| Channel | Availability |
|---------|-------------|
| Email: support@intentsolutions.io | All tiers |
| Slack channel | Premium tier only |
| Support portal (if implemented) | All tiers |

### Required Information

Support requests must include:
1. Skill name and version
2. Description of issue
3. Steps to reproduce
4. Expected vs. actual behavior
5. Environment details (SDK version, Claude Code version)
6. Severity assessment (Client's view)

### Response Process

1. Provider acknowledges receipt
2. Provider triages and assigns severity
3. Provider communicates estimated resolution
4. Provider works on resolution
5. Provider delivers fix or workaround
6. Client confirms resolution
